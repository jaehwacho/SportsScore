# PRD — 한국중고배구 여고부 경기 결과 사이트 (멀티 대회)

| 항목 | 내용 |
|---|---|
| 문서 버전 | v2.0 (멀티 대회 반영) |
| 상태 | 검토용 |
| 대상 | 한국중고배구 여고부 연맹전 경기 결과 (대회 단위로 누적) |
| 데이터 출처 | 한국중고배구연맹 / ㈜호각 |
| 연결 데이터 | `danyang_2026_data.json`, `samcheok_2026_data.json` (대회별 1파일) |
| 비고 | 본 문서는 v1.0(단양대회 단일 기준)을 대체합니다. |

---

## 1. 배경 & 목적

한국중고배구 여고부 연맹전의 경기 데이터를 PC·모바일에서 쉽게 볼 수 있는 결과 사이트로 제공한다. 데이터는 대회마다 PDF로만 배포되어 접근성·가독성이 떨어지고, 특히 모바일에서 표를 보기 어렵다.

현재까지 두 개 대회 데이터가 확보되었다: **춘계 단양대회**(3월)와 **1차 연맹전 삼척대회**(4월). 두 대회에는 천안청수·강릉여고·광주체고·제천여고 등 **여러 팀과 선수가 공통으로 출전**하며, 시즌 동안 대회가 계속 추가될 것으로 보인다. 따라서 단일 대회 페이지가 아니라, **대회를 누적해 보여주고 같은 팀·선수를 대회 간에 이어서 볼 수 있는 플랫폼**으로 설계한다.

## 2. 목표 & 성공 지표

- 여러 대회의 일정·결과·순위·선수 기록을 한 사이트에서 확인할 수 있게 한다.
- 모바일에서 가로 스크롤 없이 핵심 정보를 열람할 수 있게 한다.
- 원하는 대회·경기·팀·선수를 3번의 탭 이내로 찾을 수 있게 한다.
- 같은 팀·선수의 기록을 대회를 넘어 이어서 볼 수 있게 한다.
- (정량 예시) 모바일 첫 화면 로딩 2초 이내, 새 대회 추가 시 데이터 파일만 추가하면 노출.

## 3. 타겟 사용자

| 그룹 | 주요 니즈 |
|---|---|
| 선수·학부모 | 우리 팀·선수의 대회별 결과와 개인 기록, 시즌 흐름 확인 |
| 일반 배구 팬 | 대회별 대진표·우승팀, 주목 선수의 대회 간 활약 파악 |
| 연맹·관계자 | 공식 기록 열람 및 외부 공유용으로 정리된 화면, 시즌 누적 관리 |

## 4. 범위 (Scope)

**포함 (In scope)**

1. 대회 선택·목록 (대회를 추가·전환)
2. 대회별 경기 일정·결과 (세트 스코어 포함)
3. 대회별 대진표 (예선 → 4강 → 결승)
4. 대회별 팀 순위·종합 기록
5. 대회별 선수 개인 기록·검색
6. 같은 선수의 대회 간 기록 보기 (선수 식별 기반)

**제외 (Out of scope, v1)**

- 실시간 라이브 스코어, 영상·하이라이트, 로그인·댓글, 다국어
- 대회 간 자동 랭킹/MVP 산정(데이터 신뢰도 확보 후 검토)

## 5. 정보 구조 (IA) / 화면 구성

상단에 **대회 선택기**를 두고, 선택된 대회 안에서 일정·대진표·순위·선수로 이동한다.

| 화면 | 핵심 내용 | 드릴다운 |
|---|---|---|
| 홈 | 최신 대회 요약(우승·준우승) + 전체 대회 목록 | 대회 선택 |
| 대회 홈 | 해당 대회 개요·대진표 요약·빠른 이동 | — |
| 일정·결과 | 날짜별 경기 리스트, 세트 스코어 | 경기 상세(세트별 점수) |
| 대진표 | 예선 → 4강 → 결승 브래킷 | 해당 경기로 이동 |
| 팀 순위 | 종합 순위표(승패·세트·득실·성적) | 팀 상세 |
| 팀 상세 | 해당 대회 경기 이력 + 팀 합계 | 같은 팀의 다른 대회로 이동 |
| 선수 기록 | 검색·필터·정렬 가능한 선수 테이블 | 선수 상세 |
| 선수 상세 | 신체정보 + 누적 스탯 + **출전 대회 목록 및 대회별 비교** | 다른 대회 기록 |

## 6. 기능 요구사항

### F1. 대회 선택·목록
- 확보된 모든 대회를 일정 순으로 목록화하고, 상단 선택기로 전환한다.
- 기본 노출은 가장 최근 대회로 한다.

### F2. 경기 일정·결과 (대회별)
- 경기를 날짜·경기번호 순으로 표시하고, 양 팀명·세트 스코어·세트별 점수·라운드를 표기한다.

### F3. 대진표 (대회별)
- 예선 → 4강 → 결승 구조를 표시하고 우승팀을 강조한다. 모바일에서는 세로형으로 전환한다.

### F4. 팀 순위·종합 기록 (대회별)
- 컬럼: 순위, 팀명, 경기수, 승-패, 세트 승-패, 득점, 실점, 최종 성적. 정렬 지원, 상위권 강조.

### F5. 선수 개인 기록·검색 (대회별)
- 전 팀 선수 통합 테이블. 검색(선수명)·필터(팀·포지션·학년)·정렬(득점·공격성공·블로킹·서브에이스·디그 등).

### F6. 선수 대회 간 기록 보기 (신규)
- 같은 선수로 식별되면 선수 상세에서 **출전한 모든 대회의 기록을 나란히** 보여준다.
- 식별 규칙(아래 7-2 참조)에 따라 자동 매칭하되, 불확실한 경우는 분리해 둔다.

## 7. 데이터 모델

### 7-1. 구조
대회별로 1개 JSON 파일을 두고(`{대회}_data.json`), 코드와 분리한다. 새 대회는 파일 추가 + 인덱스 등록만으로 노출된다. 각 파일은 동일 스키마를 따른다.

| 영역 | 키 | 설명 |
|---|---|---|
| 대회 | `tournament` | id, 대회명, 일정, 출처, 데이터 고지문, 우승·준우승, 데이터 주석 |
| 순위 | `standings` | 팀별 승패·세트·득실·경기당·세트당 득실·최종성적 |
| 경기 | `matches` | 날짜·라운드·양 팀·세트 스코어·세트별 점수·승자 |
| 선수 | `players` | 신체정보 + 누적 스탯(team, jersey, name, grade, pos, height, weight, matches, sets, kills, total_attack, hit_pct, blocks, serve_aces, total_points, player_score, rcv_*, digs, played) |

상위에 대회 목록을 묶는 인덱스(`tournaments` 배열: id·이름·일정·파일경로)를 두어 F1을 지원한다.

### 7-2. 정체성(Identity) 정규화 — 멀티 대회의 핵심 과제
대회 간 연결(F6, 팀 상세 이동)을 위해 **팀·선수를 안정적인 ID로 통일**해야 한다. 현재 데이터에는 다음 불일치가 있어 정규화 규칙이 필요하다.

- **팀명 표기 차이**: 단양대회의 "천안청수"가 삼척대회에서는 "천안청수고"로 표기됨. → 팀 정규화 테이블(별칭 → 표준 팀ID) 필요.
- **선수명 표기 차이**: 천안청수 9번이 단양 "송인지" / 삼척 "송민지"로 다름(오기 추정). → 동일인 여부 확인·매핑 필요.
- **등번호 변동 가능성**: 대회 간 등번호가 바뀔 수 있으므로, 매칭 우선순위는 (정규화 팀ID + 이름) > 등번호로 한다.
- 자동 매칭이 불확실하면 **합치지 않고 분리**하여 잘못된 통합을 방지한다.

> 두 대회 공통 출전 팀: **천안청수·강릉여고·광주체고·제천여고**. 광주체고·강릉여고는 선수 명단도 대부분 동일해 대회 간 비교 가치가 크다.

## 8. 비기능 요구사항
- **반응형**: 모바일(≤480px)·태블릿·PC 대응. 넓은 표는 모바일에서 카드형/가로 스크롤.
- **성능**: 정적 사이트 기준 첫 로딩 2초 이내, 대회 추가에도 일정한 성능.
- **확장성**: 대회 N개로 늘어도 데이터 파일 추가만으로 동작.
- **접근성**: 충분한 폰트 크기·명도 대비, 표 헤더 고정.
- **브라우저**: 최신 Chrome·Safari·삼성인터넷.

## 9. 디자인 방향 (제안)
- 깔끔한 스포츠 데이터 톤(흰 배경 + 포인트 컬러 1~2개), 한글 가독성 우선.
- 대회 선택기·우승·순위 등 핵심 내비게이션과 정보를 색·아이콘으로 강조.

## 10. 제약사항 & 리스크
- **저작권 (최우선 선결 과제)**: 원본 PDF에 "연맹 데이터는 연맹의 자산이며 서면 허락 없이 복제·유통·가공·배포를 금지한다"는 고지가 있다. **공개 게시 전 한국중고배구연맹의 사용·게시 허가를 반드시 확인**한다.
- **정체성 정규화 오류**: 팀·선수 매핑 오류 시 대회 간 기록이 잘못 합쳐질 수 있음 → 7-2 규칙 적용, 불확실 시 분리.
- **데이터 정확성**: 원본 누락(예: 삼척 천안청수 이효담 누적 누락→경기별 합산 복원)·중복(삼척 김민서 중복 기재)·표기 오기 사례가 있어 게시 전 검수 필요.

## 11. 단계별 추진 (Phasing)
- **Phase 1 — 대회별 뷰**: 대회 선택기 + 대회별 일정·대진표·순위·선수(F1~F5). 두 대회로 즉시 구성 가능.
- **Phase 2 — 대회 간 연결**: 팀·선수 정체성 정규화 후 선수 대회 간 비교(F6), 팀 상세 대회 이동.
- **Phase 3(옵션)**: 시즌 누적 랭킹/주목 선수, 남고부·중등부 확대, 라이브 스코어.

## 12. 미결정 사항 (Open Questions)
- 연맹 게시 허가 여부 및 데이터 고지문 노출 범위.
- 정체성 정규화를 자동 규칙으로 둘지, 수동 매핑 테이블을 운영할지.
- 디자인 톤·브랜드 컬러(연맹 공식 가이드 유무).
